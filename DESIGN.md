# Bird Companion 设计草案

这个版本把打字鸟叫从“网页乐器”拆成一个桌面小宠物。它不是输入法，也暂时不是全局键盘监听器，而是一个可置顶、可拖动的小窗口：窗口获得焦点时，键盘节奏会驱动鸟的状态、声音反馈和身体动作。

## 反馈系统

- `curious`：慢速单字输入，使用更柔和、低密度的鸟声。
- `busy`：连续快速输入，切到更活跃、更明亮的鸟声。
- `alert`：删除键触发短促、警觉的反馈。
- `chorus`：回车触发最近输入节奏的三声小合唱。
- `sleepy`：空格触发安静停顿。

声音不是按字母直接固定映射，而是先判断“打字动作的状态”，再从对应声库里挑选样本。这样同样的键在不同节奏下会有不同反馈。

## 桌面形态

- Electron 无边框透明窗口。
- 默认置顶，可拖动。
- 默认只显示鸟形标记和树枝。
- 悬停时显示极简信息条：当前鸟种、状态、监听模式。
- 悬停按钮里的 `P` / `N` 可以在自然图鉴 Kawaii 小鸟和像素 Shimeji 小鸟之间切换。
- 悬停按钮下方显示轻量统计：累计敲击、本次会话敲击和当前 KPS。
- 默认使用焦点内监听。
- 悬停按钮里的 `G` 是全局监听开关，开启后用系统级 hook 接收键盘事件。
- 全局模式只传递按键类别和节奏，不记录文本内容。

## 身体动作

- `curious`：歪头、轻微波纹。
- `busy`：小跳、扇动感、落下一根羽毛。
- `alert`：惊跳、树枝急抖、落下多根羽毛。
- `chorus`：连续跳动，并出现两个淡淡的回声鸟影。
- `sleepy`：下沉、收拢成休息姿态。

鸟叫反馈还会在嘴边生成短促的 chirp marks，让声音反馈不只依赖外圈波纹。

## 打字统计

- `total keys`：累计敲击数，保存到本地 localStorage。
- `session`：本次打开后的敲击数，不跨会话保存。
- `kps`：近期节奏强度，使用至少 2 秒观察窗，避免刚开始输入时数值过度跳动。

统计只保存数字，不保存完整按键序列或输入内容。

## 后续方向

- 全局模式：用显式开关启用系统级键盘监听，只记录 key type 和节奏，不记录完整文本。
- 配置模式：选择“森林 / 城市 / 清晨 / 夜晚”等声音人格。
- 桌面宠物模式：可缩成只剩鸟和树枝，声音信息面板按需展开。
- 学习模式：把 BirdNET 或特征聚类结果用于“鸟群生态位”而不是简单分类。
